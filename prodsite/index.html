Powerful Features

Everything you need to build autonomous AI agents that work privately on your infrastructure.

Multi-Provider LLM

Seamlessly switch between Ollama (local), OpenAI, and Anthropic. Built-in LLM router with fallback and round-robin strategies.

Three-Tier Memory

Short-term conversation buffer, long-term persistent storage, and episodic memory for completed goals. Your agent remembers everything.

Production Hardening

Circuit breaker, rate limiting, API key auth, audit logging, input sanitization, graceful shutdown, and health checks built-in.

Plugin System

Extend LAIOS with custom tools and behaviors without modifying core code. Register tools, subscribe to events, intercept parameters.

Web UI + REST API

Full React dashboard with real-time WebSocket updates. FastAPI server with session management, chat, tool execution, and health monitoring.

Powerful CLI

Feature-rich command-line interface with interactive chat, one-shot goal execution, live task monitoring, memory search, and session management.

How It Works

LAIOS follows a clear cognitive pipeline: Reason, Plan, Execute, Reflect.

1

Reason

The Reasoner parses natural language into structured goals with intent classification and goal formulation.

2

Plan

The Planner decomposes goals into executable DAG task plans with dependency tracking and optimization.

3

Execute

The Executor invokes tools to complete tasks with retry logic, monitoring, and real-time progress tracking.

4

Reflect

The Reflector evaluates outcomes, learns from failures, and stores insights for continuous improvement.

Quick Start

Get up and running in under 5 minutes.

Step 1

Clone & Install

bash 
git clone https://github.com/ai-alon/LAIOS.git
cd LAIOS
pip install -e ".[dev,api,llm]"
Step 2

Set Up Local LLM

bash 
# Install Ollama
curl https://ollama.ai/install.sh | sh

# Pull a model
ollama pull gemma3:4b
Step 3

Start Using LAIOS

bash 
# Interactive chat
laios chat

# One-shot goal execution
laios run "find all Python files"

# Start web interface
laios serve --port 8000

Or use Docker

bash 
docker-compose up -d
# Access at http://localhost:80
